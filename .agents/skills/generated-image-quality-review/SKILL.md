# Generated Image Quality Review

Review only relevant changed assets and integrations; do not edit files.

- Inspect the actual image at representative sizes for prompt fidelity, count/spatial errors,
  anatomy, perspective, shadows, reflections, repeated textures, broken edges, illegible embedded
  text, crop, safe area, and visual consistency.
- Check whether generated imagery is appropriate for the audience and sensitive context. Assess
  recognizable people, consent, impersonation, brands, stereotypes, documentary framing, and
  medical/legal/financial use.
- Classify each image as decorative, informative, or functional and verify the corresponding alt
  text, empty-alt, caption, or action-label decision.
- Check dimensions, responsive variants, format, bytes, intrinsic size, loading priority, LCP risk,
  stable URLs, and social/search metadata where used.
- Describe available provenance signals precisely. They do not prove truth, ownership, consent, or
  an unchanged file. Visual artifacts are quality evidence, not proof of AI authorship.

Return material findings with asset/page/viewport references and a smallest proving check. State
manual-only residual risks when static checks cannot decide them. Recheck once after targeted fixes.
