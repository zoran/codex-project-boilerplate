# Native-Language Content Review

Review the target language as original writing for its audience, not as a word-for-word translation.
Do not edit files unless the owning task explicitly requests replacement copy.

Check meaning, terminology, register, locale conventions, grammar, punctuation, rhythm, inclusive
language, cultural fit, placeholders, UI constraints, links, numbers, dates, units, and consistency
with the product voice. Remove generic filler, stock transitions, needless repetition, exaggerated
claims, and mechanically uniform prose without erasing deliberate style.

Treat awkward fluency patterns as quality issues, never proof of AI authorship. Preserve documented
technical, legal, pricing, privacy, and operational commitments. If native-level confidence is
insufficient for the language, domain, or risk, say so and request the appropriate human review.

Return replacement wording for small issues or findings grouped as blocking, material, and minor.
State the language/locale, audience, voice assumptions, and any unresolved domain uncertainty.
